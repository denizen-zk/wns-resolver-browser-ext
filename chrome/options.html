Warning: The patterns below run on every link on every page. Avoid nested quantifiers (e.g. (a+)+) — poorly designed regex can freeze your browser.

JSON array of rules, each with a "pattern" (regex) and "group" (capture group number, default 1) that extracts the address from a link's href. Rules are tried in order; first valid match wins. Takes priority over the default patterns.

Example:

[
  {
    "pattern": "[?&]a=(0x[0-9a-fA-F]{40})",
    "group": 1
  }
]
